## 1. 产品概述

### 1.1 产品定位

ACEMATIC运营管理系统是一个**企业级战略执行管理平台**，帮助CEO和高管团队实现从公司战略制定到部门计划执行、再到成果考核的全流程数字化管理。

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| 战略落地 | 战略→计划→任务→执行全流程管控 |
| 任务追溯 | 任务来源可追溯，战略对齐可度量 |
| 流程规范 | 审核工作流确保内容质量 |
| 权限精细 | 四级角色权限，数据隔离 |

### 1.3 用户群体

| 角色 | 层级 | 职责 |
|------|------|------|
| CEO | L1 | 战略制定、全局管控、人事任免 |
| 高管 | L2 | 分管部门管理、计划审批 |
| 经理 | L3 | 本部门管理、任务分配审核 |
| 员工 | L4 | 个人任务管理、日志填报 |

## 2. 功能需求

### 2.1 用户认证模块

#### 2.1.1 登录功能

**功能描述**: 用户通过账号密码登录系统

**输入**:
- 用户名（必填）
- 密码（必填）

**输出**:
- 登录成功：返回Token，跳转到首页
- 登录失败：提示错误信息

**业务规则**:
- 用户名和密码不能为空
- 密码错误3次锁定账户15分钟
- Token有效期7天

#### 2.1.2 权限控制

**功能描述**: 基于角色的权限控制系统

**权限矩阵**:

| 功能 | CEO | 高管 | 经理 | 员工 |
|------|-----|------|------|------|
| 战略创建/编辑/删除 | ✅ | ✅ | ❌ | ❌ |
| 战略审核 | ✅ | ❌ | ❌ | ❌ |
| 计划创建 | ✅ | ✅ | ✅ | ❌ |
| 计划删除 | ✅ | ✅ | ❌ | ❌ |
| 任务创建/编辑 | ✅ | ✅ | ✅ | ✅ |
| 任务删除 | ✅ | ✅ | ❌ | ❌ |
| 角色权限配置 | ✅ | ❌ | ❌ | ❌ |
| 用户管理 | ✅ | ✅ | ❌ | ❌ |

### 2.2 战略管理模块

#### 2.2.1 创建战略

**功能描述**: CEO/高管创建公司年度战略

**输入字段**:
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | 是 | 战略名称 |
| 描述 | 文本 | 否 | 战略详细描述 |
| 年度 | 数字 | 是 | 适用年度 |
| 开始日期 | 日期 | 是 | 战略开始时间 |
| 结束日期 | 日期 | 是 | 战略结束时间 |

**业务规则**:
- 创建后状态为"草稿"
- 草稿可编辑、删除
- 提交审核后不可编辑

#### 2.2.2 审核工作流

**状态流转**:
```
草稿(draft) → 提交审核 → 待审核(pending) → 审核通过 → 生效(active)
                     ↓
                   审核驳回 → 草稿(draft)
```

**审核规则**:
- 只有CEO可以审核战略
- 审核通过后状态变为"生效"
- 生效后的战略不可直接编辑，需申请变更

### 2.3 计划管理模块

#### 2.3.1 计划类型

| 类型 | 创建者 | 审核者 |
|------|--------|--------|
| 公司计划 | CEO | CEO |
| 部门计划 | CEO/高管/经理 | CEO/高管 |
| 个人计划 | 所有用户 | 经理及以上 |

#### 2.3.2 变更申请

**功能描述**: 生效计划需要修改时提交变更申请

**输入字段**:
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 变更原因 | 文本 | 是 | 说明变更原因 |
| 变更内容 | JSON | 是 | 新的计划内容 |

**业务规则**:
- 只有计划所有者或管理员可申请变更
- 原审核者审核变更申请
- 审核通过后更新内容

### 2.4 任务管理模块

#### 2.4.1 任务来源

| 来源类型 | 说明 |
|----------|------|
| 上级分配 | 上级直接分配给下级 |
| 自主发起 | 员工主动创建 |
| 计划分解 | 从计划自动分解生成 |

#### 2.4.2 任务状态

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 草稿 | 创建中 | 编辑、删除、提交确认 |
| 待确认 | 等待确认 | 确认、拒绝 |
| 进行中 | 执行中 | 更新进度、提交验收 |
| 已完成 | 等待验收 | - |
| 已验收 | 验收通过 | - |

#### 2.4.3 战略对齐

**功能描述**: 任务关联上层计划，计算战略对齐度

**计算公式**: 
```
战略对齐度 = 有关联目标的任务数 / 总任务数 × 100%
```

**预警规则**:
- 🟢 对齐度 ≥ 90%：正常
- 🟡 对齐度 70%-90%：提醒改进
- 🔴 对齐度 < 70%：需提交说明

## 4. 数据模型

### 4.1 核心实体

#### User（用户）
```
{
  id: string           // 用户ID
  username: string     // 用户名
  name: string         // 姓名
  role: string         // 角色
  departmentId: string // 部门ID
  position: string     // 岗位
  email: string        // 邮箱
  phone: string        // 手机号
  isActive: boolean    // 是否启用
}
```

#### Strategy（战略）
```
{
  id: string           // 战略ID
  title: string        // 战略标题
  year: number         // 年度
  status: string       // 状态
  progress: number     // 进度
  createdById: string  // 创建者ID
}
```

#### Plan（计划）
```
{
  id: string           // 计划ID
  title: string        // 计划标题
  type: string         // 类型
  status: string       // 状态
  strategyId: string   // 关联战略
  departmentId: string // 所属部门
  ownerId: string      // 负责人
}
```

#### Task（任务）
```
{
  id: string           // 任务ID
  title: string        // 任务标题
  status: string       // 状态
  priority: string     // 优先级
  sourceType: string   // 来源类型
  assigneeId: string   // 执行人
  planId: string       // 关联计划
}
```
