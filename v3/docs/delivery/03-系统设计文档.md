### 3.3 中间件设计

#### 认证中间件 (authMiddleware)

```typescript
// 验证JWT Token
- 从Header提取Token
- 验证Token有效性
- 解码用户信息到req.user
```

#### 角色中间件 (roleMiddleware)

```typescript
// 基于角色的权限验证
- 检查用户角色是否在允许列表中
- 返回403或继续
```

#### 权限中间件 (permissionMiddleware)

```typescript
// 基于权限码的验证
- CEO默认拥有所有权限

- 检查是否拥有所需权限码
```

## 6. 工作流设计

### 6.1 审核工作流

```
┌─────────┐   submit()   ┌─────────┐   approve()   ┌─────────┐
│  Draft  │ ───────────→ │ Pending │ ────────────→ │  Active │
└─────────┘              └─────────┘               └─────────┘
     ↑                        │ reject()               │
     └────────────────────────┘                        │
     │                                                 │
     │                  changeRequest()               │
     └─────────────────────────────────────────────────┘
```

### 6.2 状态机

| 实体 | 状态 | 转换条件 |
|------|------|----------|
| 战略 | draft→pending | 提交审核 |
| 战略 | pending→active | 审核通过 |
| 战略 | pending→draft | 审核驳回 |
| 计划 | draft→pending | 提交审核 |
| 计划 | pending→active | 审核通过 |
| 任务 | draft→in_progress | 开始执行 |
| 任务 | in_progress→completed | 完成任务 |
| 任务 | completed→verified | 验收通过 |

## 7. 安全设计

### 7.1 认证机制

- JWT Token认证
- Token有效期7天
- 自动续期机制

### 7.2 密码安全

- bcrypt加密存储
- 盐值随机生成
- 密码强度校验

### 7.3 权限验证

- 前端按钮级权限控制
- 后端API权限验证
- 数据行级权限过滤

### 7.4 数据安全

- SQL注入防护(Prisma ORM)
- XSS防护(输入转义)
- CSRF防护(Token验证)
