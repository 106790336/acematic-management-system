## 3. 备份恢复

### 3.1 数据备份

```bash
#!/bin/bash
# 备份脚本 /opt/scripts/backup.sh

BACKUP_DIR="/opt/backups"
DATE=$(date +%Y%m%d_%H%M%S)
FILENAME="acematic_${DATE}.sql.gz"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 数据库备份
pg_dump -U acematic acematic_ops | gzip > $BACKUP_DIR/$FILENAME

# 上传文件备份
tar -czf $BACKUP_DIR/uploads_${DATE}.tar.gz /opt/acematic-ops/backend/uploads/

# 保留7天
find $BACKUP_DIR -name "*.gz" -mtime +7 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete

echo "Backup completed: $FILENAME"
```

### 3.2 定时备份

```bash
# 编辑 crontab
crontab -e

# 每天凌晨3点备份
0 3 * * * /opt/scripts/backup.sh >> /var/log/backup.log 2>&1
```

### 3.3 数据恢复

```bash
# 解压备份文件
gunzip acematic_20260416_030000.sql.gz

# 恢复数据库
psql -U acematic acematic_ops < acematic_20260416_030000.sql

# 恢复上传文件
tar -xzf uploads_20260416_030000.tar.gz -C /
```

```sql

SELECT * FROM pg_stat_statements ORDER BY total_time DESC LIMIT 10;

-- 创建索引
```

```bash
pm2 status
```

## 7. 告警配置

### 7.1 PM2告警

```bash
# 安装 pm2-server-monit
pm2 install pm2-server-monit

# 配置邮件告警
pm2 set pm2-server-monit:email your@email.com
```

### 7.2 系统告警脚本

```bash
#!/bin/bash
# /opt/scripts/alert.sh

CPU_THRESHOLD=80
MEM_THRESHOLD=90

# CPU告警
CPU_USAGE=$(top -bn1 | grep "Cpu(s)" | awk '{print int($2)}')
if [ $CPU_USAGE -gt $CPU_THRESHOLD ]; then
    echo "CPU usage is high: ${CPU_USAGE}%" | mail -s "CPU Alert" admin@company.com
fi

# 内存告警
MEM_USAGE=$(free | grep Mem | awk '{print int($3/$2 * 100)}')
if [ $MEM_USAGE -gt $MEM_THRESHOLD ]; then
    echo "Memory usage is high: ${MEM_USAGE}%" | mail -s "Memory Alert" admin@company.com
fi
```

## 8. 升级维护

### 8.1 升级步骤

```bash
# 1. 备份数据
/opt/scripts/backup.sh

# 2. 停止服务
pm2 stop acematic-api

# 3. 备份当前版本
cp -r /opt/acematic-ops /opt/acematic-ops.bak

# 4. 上传新版本
scp -r ./new-version/* user@server:/opt/acematic-ops/

# 5. 更新依赖
cd /opt/acematic-ops/backend
npm install

# 6. 数据库迁移
npx prisma migrate deploy

# 7. 重启服务
pm2 start acematic-api

# 8. 验证服务
curl http://localhost:3000/api/health
```

### 8.2 回滚步骤

```bash
# 1. 停止服务
pm2 stop acematic-api

# 2. 恢复代码
rm -rf /opt/acematic-ops
mv /opt/acematic-ops.bak /opt/acematic-ops

# 3. 恢复数据库
gunzip -c /opt/backups/last_backup.sql.gz | psql -U acematic acematic_ops

# 4. 重启服务
pm2 start acematic-api
```
