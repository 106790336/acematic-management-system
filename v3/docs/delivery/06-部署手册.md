## 2. 部署步骤

### 2.1 环境准备

```bash
# 1. 安装 Node.js
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
sudo yum install -y nodejs

# 2. 安装 PostgreSQL
sudo yum install -y postgresql-server postgresql-contrib
sudo postgresql-setup initdb
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 3. 安装 Nginx
sudo yum install -y nginx
sudo systemctl start nginx
sudo systemctl enable nginx

# 4. 安装 PM2
sudo npm install -g pm2
```

### 2.2 数据库配置

```bash
# 1. 创建数据库用户
sudo -u postgres psql
CREATE USER acematic WITH PASSWORD 'your_password';
CREATE DATABASE acematic_ops OWNER acematic;
GRANT ALL PRIVILEGES ON DATABASE acematic_ops TO acematic;
\q

# 2. 配置数据库连接
# 修改 postgresql.conf
sudo vi /var/lib/pgsql/data/postgresql.conf
# 取消注释 listen_addresses = 'localhost'

# 修改 pg_hba.conf
sudo vi /var/lib/pgsql/data/pg_hba.conf
# 添加: local all acematic md5

# 重启数据库
sudo systemctl restart postgresql
```

### 2.3 项目部署

```bash
# 1. 创建项目目录
sudo mkdir -p /opt/acematic-ops
sudo chown $USER:$USER /opt/acematic-ops

# 2. 上传部署包
scp -i ~/.ssh/deploy_key acematic-ops-deploy.tar.gz user@server:/opt/

# 3. 解压并安装
cd /opt
tar -xzvf acematic-ops-deploy.tar.gz
cd acematic-ops

# 4. 安装依赖
cd backend && npm install
cd ../frontend && npm install

# 5. 配置环境变量
cd ../backend
cp .env.example .env
vi .env
# 配置数据库连接:
# DATABASE_URL="postgresql://acematic:password@localhost:5432/acematic_ops?schema=public"
# JWT_SECRET="your-secret-key"

# 6. 初始化数据库
npx prisma generate
npx prisma migrate deploy
npx tsx prisma/seed.ts

# 7. 构建前端
cd ../frontend
npm run build
```

### 2.4 启动服务

```bash
# 1. 启动后端
cd /opt/acematic-ops/backend
pm2 start npm --name "acematic-api" -- run start
pm2 save
pm2 startup

# 2. 配置 Nginx
sudo vi /etc/nginx/conf.d/acematic.conf
```

**Nginx 配置**:
```nginx
server {
    listen 80;
    server_name okr.acematic.com.cn;

    # 前端静态文件
    location / {
        root /opt/acematic-ops/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
# 测试并重启 Nginx
sudo nginx -t
sudo systemctl restart nginx
```

## 3. HTTPS 配置

### 3.1 使用 Let's Encrypt

```bash
# 1. 安装 certbot
sudo yum install -y certbot python3-certbot-nginx

# 2. 获取证书
sudo certbot --nginx -d okr.acematic.com.cn

# 3. 设置自动续期
sudo crontab -e
# 添加: 0 3 * * * /usr/bin/certbot renew --quiet
```

### 3.2 手动配置证书

```nginx
server {
    listen 443 ssl;
    server_name okr.acematic.com.cn;

    ssl_certificate /etc/nginx/ssl/okr.acematic.com.cn.crt;
    ssl_certificate_key /etc/nginx/ssl/okr.acematic.com.cn.key;

    # ... 其他配置
}

server {
    listen 80;
    server_name okr.acematic.com.cn;
    return 301 https://$server_name$request_uri;
}
```

## 4. 数据备份

### 4.1 自动备份脚本

```bash
#!/bin/bash
# /opt/scripts/backup.sh

BACKUP_DIR="/opt/backups"
DATE=$(date +%Y%m%d_%H%M%S)
FILENAME="acematic_backup_${DATE}.sql.gz"

mkdir -p $BACKUP_DIR

# 数据库备份
pg_dump -U acematic acematic_ops | gzip > $BACKUP_DIR/$FILENAME

# 保留最近7天的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

echo "Backup completed: $FILENAME"
```

### 4.2 设置定时备份

```bash
# 添加到 crontab
crontab -e
# 每天凌晨3点执行备份
0 3 * * * /opt/scripts/backup.sh >> /var/log/backup.log 2>&1
```

## 6. 故障排查

### 6.1 服务无法启动

```bash
# 查看详细日志
pm2 logs acematic-api --lines 100

# 检查端口占用
netstat -tlnp | grep 3000

# 检查环境变量
cat /opt/acematic-ops/backend/.env
```

### 6.2 数据库连接失败

```bash
# 检查 PostgreSQL 状态
systemctl status postgresql

# 测试连接
psql -U acematic -d acematic_ops -h localhost

# 检查权限
sudo vi /var/lib/pgsql/data/pg_hba.conf
```

### 6.3 Nginx 错误

```bash
# 检查配置
sudo nginx -t

# 查看错误日志
tail -f /var/log/nginx/error.log
```
