## 一、产品概述

### 1.1 产品定位
运营管理系统是一套面向中小企业管理层的战略执行工具，覆盖"战略解码→目标分解→执行跟踪→复盘调整"全流程，实现经营数据的可视化、执行过程的可追踪、管理决策的数据化。

### 1.2 核心价值
- **战略落地**：将年度目标逐层分解到月度计划、周计划，确保战略不悬空
- **过程透明**：实时掌握各中心经营状况，问题早发现、早处理
- **数据驱动**：自动生成分析报告，辅助管理决策
- **闭环管理**：计划-执行-检查-改进（PDCA）全流程闭环

### 1.3 目标用户
| 用户角色 | 用户画像 | 核心需求 | 使用频率 |
|----------|----------|----------|----------|
| 总经理 | 企业决策者 | 掌握整体经营状况、重大问题预警 | 每日查看驾驶舱，每周参加周例会，每月审批月度报告 |
| 运营总监 | 战略执行负责人 | 目标分解、过程跟踪、数据分析、报告生成 | 每日操作系统，核心用户 |
| 中心负责人 | 营销/产品/运营中心负责人 | 提交计划、汇报进度、跟踪问题 | 每周提交周报，每月提交月度计划 |
| 普通员工 | 执行层 | 接收任务、反馈进度 | 按需使用 |

### 1.4 系统规模
- 用户数：30-50人
- 数据量：年增约5000条记录
- 并发量：低（内部管理系统）

#### 模块2：战略管理

##### 2.1 年度目标管理

**功能描述**：管理三个中心的年度经营目标，支持目标设定、分解、跟踪、调整。

**数据结构**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 年度 | 数字 | 2026 |
| 中心名称 | 单选 | 营销中心/产品中心/运营中心 |
| 指标名称 | 文本 | 如"年度营收目标" |
| 指标类型 | 单选 | 财务类/客户类/流程类/学习成长类 |
| 目标值 | 数字 | 5000万 |
| 权重 | 数字 | 30% |
| 一季度目标 | 数字 | 自动计算（年度÷4） |
| 二季度目标 | 数字 | 自动计算 |
| 三季度目标 | 数字 | 自动计算 |
| 四季度目标 | 数字 | 自动计算 |
| 实际达成 | 数字 | 年底填写 |
| 达成率 | 计算 | 实际÷目标×100% |
| 绩效得分 | 计算 | 达成率×权重 |
| 状态 | 单选 | 草稿/已签订/执行中/已完成 |

**页面设计**：
- **列表视图**：表格形式展示所有目标
- **卡片视图**：按中心分组，卡片形式展示
- **甘特图视图**：展示年度目标时间轴

**操作流程**：
1. 运营总监创建年度目标模板
2. 各中心负责人填写本中心目标
3. 总经理审核确认
4. 打印签字版，上传系统
5. 年底填写实际达成，系统自动计算绩效

##### 2.2 目标进度跟踪

**功能描述**：实时跟踪年度目标完成进度，自动计算达成率。

**计算逻辑**：
- 月度达成率 = 本月实际 ÷ 本月目标 × 100%
- 年度达成率 = 累计实际 ÷ 年度目标 × 100%
- 预测达成率 = (累计实际 + 剩余月数×近3月平均) ÷ 年度目标 × 100%

**可视化呈现**：
- 进度条：直观展示完成进度
- 趋势图：展示逐月累计进度
- 预警标识：达成率<85%显示红色，85%-95%显示黄色，≥95%显示绿色

#### 模块3：计划管理

##### 3.1 月度经营计划

**功能描述**：各中心每月提交经营计划，运营总监审核后执行。

**数据结构**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 年月 | 日期 | 2026-04 |
| 中心名称 | 单选 | 营销中心/产品中心/运营中心 |
| 提交人 | 成员 | 自动获取当前用户 |
| 计划事项 | 文本 | 具体任务描述 |
| 目标值 | 文本 | 可量化的目标 |
| 责任人 | 成员 | 任务责任人 |
| 预算 | 数字 | 预计费用 |
| 完成时限 | 日期 | 截止日期 |
| 优先级 | 单选 | 高/中/低 |
| 审核状态 | 单选 | 待提交/待审核/已通过/需修改 |
| 审核意见 | 文本 | 运营总监填写 |
| 执行状态 | 单选 | 未开始/进行中/已完成/延期 |

**操作流程**：
```
中心负责人提交（每月25日前）
    ↓
运营总监审核（每月27日前）
    ↓
    ├── 通过 → 进入执行阶段
    └── 驳回 → 修改后重新提交
    ↓
月末复盘（对照计划检查执行情况）
```

##### 3.2 周计划分解

**功能描述**：将月度计划分解到每周，细化执行节奏。

**数据结构**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 周次 | 日期 | 选择周一日期 |
| 关联月度计划 | 关联 | 关联到月度计划项 |
| 本周目标 | 文本 | 本周要完成的目标 |
| 关键动作 | 文本 | 具体执行动作 |
| 预期产出 | 文本 | 预期的结果 |
| 完成情况 | 文本 | 周末填写 |

##### 3.3 计划执行跟踪

**功能描述**：实时跟踪计划执行情况，自动预警延期风险。

**预警规则**：
- 距完成时限3天，状态≠已完成 → 发送提醒
- 超过完成时限，状态≠已完成 → 标记延期，通知运营总监
- 同一计划连续延期2次 → 标记高风险，通知总经理

#### 模块5：分析复盘

##### 5.1 月度经营分析

**功能描述**：每月汇总经营数据，自动生成分析报告。

**数据结构**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 年月 | 日期 | 2026-04 |
| 中心名称 | 单选 | 营销中心/产品中心/运营中心 |
| 指标名称 | 文本 | 关联到年度目标 |
| 本月目标 | 数字 | 从月度计划自动获取 |
| 本月实际 | 数字 | 手动填写 |
| 达成率 | 计算 | 实际÷目标×100% |
| 去年同期 | 数字 | 历史数据 |
| 同比增长 | 计算 | (本月-去年)÷去年×100% |
| 上月实际 | 数字 | 历史数据 |
| 环比增长 | 计算 | (本月-上月)÷上月×100% |
| 偏差原因 | 多行文本 | 达成率异常需填写 |
| 改进措施 | 多行文本 | 具体措施 |

**AI分析功能**：
- 自动生成月度经营分析报告
- 偏差原因智能分析（使用5Why分析法）
- 改进措施智能建议

##### 5.2 偏差分析

**功能描述**：对重大偏差进行深度分析，定位根本原因。

**分析框架**：
- 5Why分析法：连续追问5个"为什么"
- 鱼骨图分析：从人、机、料、法、环、测六个维度分析
- 帕累托分析：识别导致80%问题的20%原因

##### 5.3 改进措施跟踪

**功能描述**：跟踪改进措施的执行效果。

**数据结构**：
| 字段 | 类型 | 说明 |

| 关联问题 | 关联 | 关联到问题清单 |
| 措施描述 | 多行文本 | 具体改进措施 |
| 责任人 | 成员 | 措施执行人 |
| 计划完成时间 | 日期 | 预计完成时间 |
| 执行进度 | 数字 | 0-100% |
| 效果验证 | 文本 | 措施执行后的效果 |
| 是否有效 | 单选 | 有效/部分有效/无效 |

## 三、技术架构设计

### 3.1 技术栈选择

#### 前端技术栈
- **框架**：Vue 3 + TypeScript
- **UI组件库**：Element Plus（PC端）或 Ant Design Vue
- **状态管理**：Pinia
- **路由**：Vue Router
- **HTTP请求**：Axios
- **图表库**：ECharts
- **构建工具**：Vite

#### 后端技术栈
- **框架**：Python Flask 或 FastAPI
- **数据库**：SQLite（开发/小规模）或 PostgreSQL（生产环境）
- **ORM**：SQLAlchemy
- **认证**：JWT
- **API文档**：Swagger/OpenAPI

#### 部署方案
- **前端**：Vercel 或 Netlify（免费托管）
- **后端**：Railway 或 Render（免费层）
- **数据库**：Supabase（免费PostgreSQL）或 本地SQLite

#### AI能力集成
- **大模型API**：豆包API、智谱API、或本地部署Ollama
- **应用场景**：周报摘要、报告生成、偏差分析

### 3.4 API接口设计

```
POST /api/auth/login          # 用户登录

GET  /api/auth/profile        # 获取当前用户信息

GET    /api/goals             # 获取年度目标列表

PUT    /api/plans/:id         # 更新计划

```

#### 问题清单接口

GET    /api/issues            # 获取问题列表

GET    /api/issues/alerts     # 获取预警问题

## 四、非功能性需求

### 4.1 性能要求
- 页面加载时间：< 3秒
- API响应时间：< 500ms（95%请求）

- 用户密码加密存储（bcrypt）
- API请求需要JWT认证
- 敏感操作需要二次确认
- 数据库定期备份

- 系统可用性：> 99%（工作日8:00-20:00）
- 数据备份：每日自动备份
- 故障恢复：< 4小时

### 4.4 兼容性要求
- 浏览器：Chrome、Edge、Firefox最新版
- 屏幕分辨率：≥1366×768
- 移动端：支持企业微信内置浏览器
