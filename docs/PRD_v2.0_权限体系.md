**版本：v2.0 最终版**  
**日期：2026-04-12**  
**更新说明：新增完整权限控制体系 + 任务来源与战略对齐机制**

## 一、系统概述

### 1.1 定位
为ACEMATIC（年营收4000万，12人团队）打造的闭环运营管理系统，实现"战略→计划→任务→执行"全链路可控、可追踪、可复盘。

### 1.2 核心设计原则
1. **权限精准控制** - 任务只有分配到的人才能操作
2. **来源唯一追溯** - 每个任务必须有明确来源（上级分配/主动申请/战略分解）
3. **战略对齐闭环** - 战略→计划→任务→执行→复盘完整关联
4. **零复制执行** - AI全权执行，系统自动化管理

## 二、权限控制体系

### 2.1 权限模型

#### 2.1.1 角色定义

| 角色 | 职责范围 | 可查看数据 |
|------|----------|-----------|
| **总经理** | 全公司战略与运营 | 所有数据 |
| **运营总监** | 运营中心管理 | 运营中心+下属部门 |
| **营销总监** | 营销中心管理 | 营销中心+下属部门 |
| **产品总监** | 产品中心管理 | 产品中心+下属部门 |
| **中心负责人** | 本中心人员与任务 | 本中心数据 |
| **部门经理** | 本部门人员与任务 | 本部门数据 |
| **普通员工** | 个人任务执行 | 仅限本人相关任务 |

#### 2.1.2 任务权限矩阵

```
权限维度：
┌─────────────────────────────────────────────────────────────┐
│                    任务操作权限                               │
├─────────────────────────────────────────────────────────────┤
│  查看权限    │ 谁可以查看任务详情                             │
│  编辑权限    │ 谁可以修改任务内容                             │
│  进度更新    │ 谁可以更新任务进度                             │
│  进度描述    │ 谁可以添加进度描述/备注                        │
│  审批权限    │ 谁可以审批任务申请/验收                        │
│  删除权限    │ 谁可以删除任务                                 │
│  分配权限    │ 谁可以给他人分配任务                           │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 任务权限规则详解

#### 2.2.1 任务创建者（分配者）

| 操作 | 权限 | 说明 |
|------|------|------|
| 查看任务 | ✅ | 可查看自己创建/分配的所有任务 |
| 编辑任务 | ✅ | 可修改任务标题、截止时间、优先级（执行前） |
| 进度更新 | ❌ | 不可代为更新（需执行者本人操作） |
| 进度描述 | ✅ | 可添加指导/反馈描述 |
| 审批任务 | ✅ | 审批主动申请的任务 |
| 验收任务 | ✅ | 验收已完成的任务 |
| 删除任务 | ✅ | 任务未开始前可删除 |
| 重新分配 | ✅ | 可转派给他人 |

#### 2.2.2 任务执行者（被指派人）

| 操作 | 权限 | 说明 |
|------|------|------|
| 查看任务 | ✅ | 仅可查看分配给自己的任务 |
| 编辑任务 | ❌ | 不可修改任务定义（可备注说明） |
| 进度更新 | ✅ | **唯一可更新进度的人** |
| 进度描述 | ✅ | **唯一可添加执行描述的人** |
| 提交完成 | ✅ | 提交完成待验收 |
| 申请延期 | ✅ | 可申请延期并说明原因 |

#### 2.2.3 上级领导（审批者）

| 操作 | 权限 | 说明 |
|------|------|------|
| 查看任务 | ✅ | 可查看下属所有任务 |
| 审批任务 | ✅ | 审批主动申请的任务 |
| 验收任务 | ✅ | 验收下属提交完成的任务 |
| 进度反馈 | ✅ | 可点评任务进度/质量 |

#### 2.2.4 同级/无关人员

| 操作 | 权限 | 说明 |
|------|------|------|
| 查看任务 | ❌ | 不可查看他人任务 |
| 任何操作 | ❌ | 无任何操作权限 |

### 2.4 数据权限控制

#### 2.4.1 任务可见性规则

```
任务可见性判断：
IF 用户 = 总经理 → 所有任务可见
IF 用户 = 任务创建者 → 该任务可见
IF 用户 = 任务执行者 → 该任务可见
IF 用户 = 任务执行者的直属上级 → 该任务可见
IF 用户 = 任务执行者的中心负责人 → 该任务可见
ELSE → 任务不可见
```

#### 2.4.2 敏感信息脱敏

| 信息类型 | 员工本人 | 直属上级 | 中心负责人 | 总经理 |
|---------|---------|---------|-----------|--------|
| 任务详情 | ✅完整 | ✅完整 | ✅完整 | ✅完整 |
| 执行描述 | ✅完整 | ✅完整 | ✅完整 | ✅完整 |
| 延期原因 | ✅完整 | ✅完整 | ✅完整 | ✅完整 |
| 绩效评分 | ✅本人 | ✅下属 | ✅本中心 | ✅全部 |
| 薪资关联 | ✅本人 | ❌ | ❌ | ✅全部 |

## 三、任务来源与战略对齐机制

### 3.1 任务来源体系

所有工作任务必须有明确来源，系统禁止创建无来源任务：

```
┌─────────────────────────────────────────────────────────────┐
│                    任务来源类型                              │
├─────────────────────────────────────────────────────────────┤
│  来源1: 上级分配 (assigned)                                  │
│    ├── 总经理分配给运营总监                                  │
│    ├── 运营总监分配给中心负责人                              │
│    └── 中心负责人分配给部门成员                              │
│    【权限】分配者创建，执行者确认接收                        │
├─────────────────────────────────────────────────────────────┤
│  来源2: 主动提交 + 上级确认 (self_initiated)                 │
│    ├── 员工提交任务申请                                      │
│    └── 上级审核确认后生效                                    │
│    【权限】执行者申请，上级审批通过后方可操作                │
├─────────────────────────────────────────────────────────────┤
│  来源3: 战略计划分解 (plan_decomposition)                    │
│    ├── 年度战略 → 年度计划                                   │
│    ├── 年度计划 → 季度计划                                   │
│    ├── 季度计划 → 月度计划                                   │
│    └── 月度计划 → 周/日任务                                  │
│    【权限】系统自动关联，执行者按分解任务执行                │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 战略对齐机制

```
                    年度战略
                        │
                        ▼ 拆解
                年度目标（营收/利润/客户等）
                        │
                        ▼ 分解
                季度里程碑
                        │
                        ▼ 细化
                月度经营计划
                        │
                        ▼ 执行分解
                周计划 / 每日任务
                        │
                        ▼ 关联对齐
                个人工作任务池
```

**对齐校验规则**：
- 每项任务必须关联至少一个上层计划/目标
- 系统自动计算"战略对齐度" = 有关联目标的任务数 ÷ 总任务数
- 未对齐任务需说明理由，否则纳入绩效考核扣分项

### 3.3 战略对齐度计算

```
个人战略对齐度 = 有明确战略关联的任务数 ÷ 个人总任务数 × 100%

中心战略对齐度 = 中心内各成员对齐度加权平均
公司战略对齐度 = 所有中心对齐度加权平均

预警规则：
🟢 对齐度 ≥ 90%：正常
🟡 对齐度 70%-90%：提醒改进
🔴 对齐度 < 70%：需提交说明
```

#### 4.2.2 任务状态流转与权限

| 状态 | 可执行操作 | 操作人 |
|------|-----------|--------|
| **draft** 草稿 | 编辑、删除、发布 | 创建者 |
| **pending** 待确认 | 确认接收、拒绝、转派 | 执行者/创建者 |
| **in_progress** 进行中 | 更新进度、添加描述、申请延期、提交完成 | 执行者 |
| **paused** 已暂停 | 恢复执行、取消 | 创建者 |
| **completed** 已完成 | 验收通过、退回修改 | 创建者 |
| **approved** 已通过 | 归档、复盘 | 创建者 |
| **rejected** 已拒绝 | 重新分配、关闭 | 创建者 |
| **cancelled** 已取消 | 恢复、删除 | 创建者 |

## 六、API权限控制

### 6.1 权限校验中间件

```
所有任务相关API必须校验：
1. 用户是否登录
2. 用户是否有该任务的查看权限
3. 对于写操作：用户是否有该任务的写权限
4. 记录权限审计日志

返回码：
• 200 - 操作成功
• 403 - 无权限（权限不足）
• 404 - 任务不存在或无查看权限
• 422 - 参数错误
```

### 6.2 关键API权限

| API | 方法 | 权限要求 | 说明 |
|-----|------|---------|------|
| /api/tasks | GET | 查看权限 | 获取任务列表（仅可见的任务）|
| /api/tasks/:id | GET | 查看权限 | 获取任务详情 |
| /api/tasks | POST | 分配权限 | 创建新任务 |
| /api/tasks/:id | PUT | 编辑权限 | 更新任务（创建者可编辑，执行者不可编辑定义）|
| /api/tasks/:id/progress | POST | 执行者权限 | **仅执行者可更新进度** |
| /api/tasks/:id/complete | POST | 执行者权限 | **仅执行者可提交完成** |
| /api/tasks/:id/approve | POST | 审批权限 | 审批任务申请 |
| /api/tasks/:id/accept | POST | 执行者权限 | **仅执行者可确认接收** |
| /api/tasks/:id/reject | POST | 执行者权限 | **仅执行者可拒绝任务** |

## 七、验收标准

### 7.1 功能验收

#### 权限控制
- [ ] 任务创建后，非执行者无法更新进度
- [ ] 任务创建后，非执行者无法添加进度描述
- [ ] 同级人员无法查看他人任务
- [ ] 任务未确认前，执行者无法操作
- [ ] 权限变更实时生效

#### 任务来源
- [ ] 任务来源必须明确，系统不允许创建无来源任务
- [ ] 上级分配任务需被分配人确认接收
- [ ] 主动提交任务需上级审批通过后方可生效
- [ ] 计划分解任务自动关联，无需手动选择

#### 战略对齐
- [ ] 战略对齐度实时计算并展示
- [ ] 对齐度低于70%触发预警
- [ ] 未对齐任务需说明理由

### 7.2 安全验收
- [ ] 权限审计日志完整记录
- [ ] 越权操作被阻止并记录
- [ ] 敏感信息仅授权人员可见
- [ ] 数据权限按组织架构隔离

### 7.3 性能验收
- [ ] 任务列表加载 < 2秒
- [ ] 权限校验 < 100ms
- [ ] 并发用户支持 ≥ 50人
