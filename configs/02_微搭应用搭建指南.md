# ACEMATIC 腾讯云微搭自动化管理系统 - 应用搭建指南

## 一、环境准备

### 1.1 腾讯云账号准备
1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 完成实名认证（企业认证）
3. 开通微搭低代码平台服务
4. 创建新应用空间：命名"ACEMATIC管理系统"

### 1.2 微搭平台入口
- 访问地址：https://weda.cloud.tencent.com/
- 选择已创建的"ACEMATIC管理系统"空间

### 1.3 开发环境配置
- 浏览器：Chrome 90+ 或 Edge 90+
- 网络：保证稳定的互联网连接
- 权限：管理员权限

---

## 二、应用创建流程

### 2.1 创建应用框架

#### 步骤1：创建主应用
1. 进入微搭控制台 → 应用管理 → 新建应用
2. 选择"模型应用"（自动生成增删改查）
3. 应用名称：ACEMATIC管理系统
4. 应用标识：ACEMATIC_MGT
5. 创建完成后进入应用编辑器

#### 步骤2：创建子应用
在应用内创建以下页面：

| 页面名称 | 页面标识 | 页面类型 | 功能说明 |
|----------|----------|----------|----------|
| 首页看板 | index | 自定义页面 | 董事长驾驶舱 |
| 目标管理 | goal_mgmt | 模型页面 | 目标录入与追踪 |
| 绩效考核 | performance | 模型页面 | 绩效评分与工资 |
| 质量管控 | quality | 模型页面 | 质量检验记录 |
| 数据报表 | report | 自定义页面 | 各类报表中心 |

## 三、数据模型配置

### 3.1 创建数据模型

#### 3.1.1 组织架构模型
1. 进入"数据模型" → "新建数据模型"
2. 模型名称：组织架构
3. 模型标识：org_structure
4. 添加字段（按数据模型文档）：

```
字段配置：
- id: 自增主键（系统自动创建）
- center_name: 文本，必填
- center_code: 文本，必填
- position_name: 文本，必填
- position_code: 文本，必填，唯一
- person_name: 文本，必填
- person_id: 文本，必填，唯一
- mobile: 文本
- salary_fixed: 数字，默认0
- salary_manage: 数字，默认0
- salary_business: 数字，默认0
- status: 枚举（在职/离职），默认在职
- create_time: 日期时间，默认当前时间
```

#### 3.1.2 KPI指标库模型
1. 模型名称：KPI指标库
2. 模型标识：kpi_indicator
3. 添加字段：

```
字段配置：
- indicator_name: 文本，必填
- indicator_code: 文本，必填，唯一
- center_code: 关联→组织架构.center_code
- position_code: 关联→组织架构.position_code
- indicator_type: 枚举（管理绩效/经营绩效）
- weight: 数字，必填
- target_value: 数字
- unit: 文本
- formula: 长文本
- data_source: 文本
- status: 枚举（启用/禁用），默认启用
```

#### 3.1.3 绩效评分模型
1. 模型名称：绩效评分
2. 模型标识：performance_score
3. 添加字段：

```
字段配置：
- year_month: 文本，必填（格式：2026-04）
- person_id: 关联→组织架构.person_id
- indicator_code: 关联→KPI指标库.indicator_code
- target_value: 数字
- actual_value: 数字
- score: 数字（系统自动计算）
- weight: 数字
- weighted_score: 数字（系统自动计算）
- data_source: 文本
- record_time: 日期时间
```

### 3.2 配置数据关联

在微搭中配置表间关系：

1. **绩效评分 → 组织架构**
   - 类型：一对多
   - 主表：组织架构
   - 从表：绩效评分
   - 关联字段：person_id

2. **绩效评分 → KPI指标库**
   - 类型：一对多
   - 主表：KPI指标库
   - 从表：绩效评分
   - 关联字段：indicator_code

3. **工资计算 → 组织架构**
   - 类型：一对多
   - 主表：组织架构
   - 从表：工资计算
   - 关联字段：person_id

## 五、工作流配置

### 5.1 每日自动化工作流

#### 步骤1：创建工作流
1. 进入"工作流" → "新建工作流"
2. 名称：每日数据同步与报告生成
3. 触发方式：定时触发
4. 触发时间：每天 08:00

#### 步骤2：配置工作流节点

```
节点1：数据同步（自定义JS）
- 调用速达ERP API获取昨日数据
- 调用企业微信API获取考勤数据
- 将数据写入绩效评分表

节点2：绩效计算（自定义JS）
- 读取本月所有绩效数据
- 按公式计算各项得分
- 更新工资计算表

节点3：预警检测（条件分支）
- 检测回款率<95%
- 检测合格率<99%
- 检测绩效<80分
- 生成预警记录

节点4：报告生成（自定义JS）
- 汇总昨日数据
- 生成HTML格式的简报

节点5：消息推送（企微消息）
- 发送每日简报给董事长
- 发送预警通知给相关人员
```

#### 步骤3：JS代码示例

**节点1：数据同步代码**
```javascript
// 调用速达ERP API获取销售数据
async function syncSudaData() {
  const apiUrl = 'https://api.suda.com/sales/daily';
  const response = await fetch(apiUrl, {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer ' + context.env.SUDA_API_KEY,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      date: context.currentDate
    })
  });
  
  const data = await response.json();
  
  // 将数据写入绩效评分表
  for (let item of data) {
    await context.model.performance_score.create({
      year_month: context.currentMonth,
      indicator_code: 'YX_KPI_001',
      actual_value: item.salesAmount,
      data_source: '速达ERP'
    });
  }
}
```

**节点2：绩效计算代码**
```javascript
// 计算绩效得分
async function calculatePerformance() {
  // 获取本月所有人员
  const persons = await context.model.org_structure.find({
    status: '在职'
  });
  
  for (let person of persons) {
    // 获取该人员的所有KPI指标
    const kpis = await context.model.kpi_indicator.find({
      position_code: person.position_code
    });
    
    let manageScore = 0;
    let businessScore = 0;
    
    for (let kpi of kpis) {
      // 获取实际完成值
      const score = await context.model.performance_score.findOne({
        person_id: person.person_id,
        indicator_code: kpi.indicator_code,
        year_month: context.currentMonth
      });
      
      // 计算得分
      let achieveRate = score.actual_value / score.target_value;
      let itemScore = 0;
      
      if (achieveRate >= 1) {
        itemScore = 100;
      } else if (achieveRate >= 0.8) {
        itemScore = achieveRate * 100;
      } else {
        itemScore = 0;
      }
      
      // 累加绩效
      if (kpi.indicator_type === '管理绩效') {
        manageScore += itemScore * kpi.weight;
      } else {
        businessScore += itemScore * kpi.weight;
      }
    }
    
    // 更新工资计算表
    await context.model.salary_calculate.update({
      person_id: person.person_id,
      year_month: context.currentMonth
    }, {
      manage_score: manageScore,
      business_score: businessScore,
      manage_actual: person.salary_manage * (manageScore / 100),
      business_actual: person.salary_business * (businessScore / 100)
    });
  }
}
```

### 5.2 一票否决工作流

#### 触发条件：质量检验记录更新

```
节点1：条件判断
- 检验结果 = '不合格'

节点2：锁定操作
- 设置veto_flag = '是'

- 如果是成品：锁定该批次成品

节点3：消息通知
- 发送《停工/停发货通知》给相关部门
- 发送预警给质量总监和总经理

节点4：记录日志
- 创建一票否决记录
```

### 5.3 月度绩效计算工作流

#### 触发条件：每月1日 00:00

```
节点1：数据准备
- 汇总上月所有数据

- 计算所有人员经营绩效

节点4：审批流程
```

```yaml
连接器名称: 速达ERP

  Authorization: Bearer {{API_KEY}}
```

```yaml

响应示例:
  {
    "code": 200,
    "data": [

        "salesAmount": 100000,
        "returnAmount": 95000,
        "orderCount": 5
```
