# ACEMATIC 腾讯云微搭自动化管理系统 - 数据模型设计文档

## 一、系统概述

### 1.1 系统架构
```
┌─────────────────────────────────────────────────────────────┐
│                    ACEMATIC 管理系统                         │
├─────────────────────────────────────────────────────────────┤
│  前端层：WEB管理后台 + 手机端H5 + 企业微信工作台             │
├─────────────────────────────────────────────────────────────┤
│  应用层：目标管理 | 绩效考核 | 质量管控 | 数据报表           │
├─────────────────────────────────────────────────────────────┤
│  数据层：微搭数据库 | 速达ERP API | 企业微信                 │
├─────────────────────────────────────────────────────────────┤
│  自动化层：工作流引擎 | 定时任务 | 消息推送                   │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 数据集成关系
```
速达ERP ──API──→ 微搭数据库 ──→ 应用层 ──→ 企微消息推送
企业微信 ──API──↗              ↖ 数据报表
```

**绩效得分计算规则**：
```
得分计算逻辑：
- 达成率 ≥ 100%：得分 = 100%（封顶150%）
- 80% ≤ 达成率 < 100%：得分 = 达成率
- 达成率 < 80%：得分 = 0

实发计算：
- 管理绩效实发 = 管理绩效基数 × (管理绩效得分/100)
- 经营绩效实发 = 经营绩效基数 × (经营绩效得分/100)
- 经营绩效最高上浮至150%（即最高9000元）

一票否决：
- 触发一票否决条件，经营绩效全额归零
```
