# ACEMATIC 腾讯云微搭 - 企业微信集成配置说明

## 一、集成概述

### 1.1 集成目标
- 实现企业微信与微搭系统的数据互通
- 每日自动推送经营简报、预警通知、工资条
- 实现考勤数据自动同步
- 在企业微信工作台嵌入微搭应用

### 1.2 集成架构
```
企业微信
  │
  │ 应用消息推送
  ▼
微搭消息中心
  │
  │ 数据拉取
  ▼
企业微信API
  │
  │ 数据同步
  ▼
微搭数据库
```

### 1.3 集成功能清单

| 功能 | 说明 | 实现方式 |
|------|------|----------|
| 考勤同步 | 每日自动同步打卡/请假数据 | 企业微信HR API |
| 审批同步 | 请假/报销/采购审批数据 | 企业微信审批API |
| 消息推送 | 简报/预警/工资条推送 | 企业微信消息API |
| 工作台应用 | 在企微内嵌入微搭H5 | 企业微信应用 |

## 二、企业微信配置

### 2.1 准备工作

#### 步骤1：企业微信管理员登录
1. 登录企业微信管理后台：https://work.weixin.qq.com/
2. 使用管理员账号登录

#### 步骤2：获取企业信息
在"我的企业"中获取：
```
企业ID: wwxxxxxxxxxxxxxxx
企业名称: ACEMATIC公司
```

### 2.2 创建自建应用

#### 步骤1：创建应用
1. 进入"应用管理" → "自建应用"
2. 点击"创建应用"
3. 填写应用信息：

```
应用名称: ACEMATIC管理系统
应用描述: 公司自动化管理平台
应用logo: 上传公司Logo（建议尺寸：108*108px）
```

#### 步骤2：设置应用主页
1. 进入应用详情页面
2. 点击"设置应用主页"
3. 选择"网页链接"
4. 输入H5应用地址：
```
https://weda.cloud.tencent.com/h5/xxxxxxxx
```
（该地址在微搭启用H5端后获取）

#### 步骤3：配置可见范围
1. 点击"可见范围"
2. 选择"全部成员"或指定部门/人员

#### 步骤4：获取应用凭证
在"凭证与回调"页面获取：
```
AgentId: 1000001
Secret: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 2.3 配置API权限

#### 步骤1：开启API接口
在"API权限"中申请以下权限：

| 权限名称 | 权限说明 | 申请理由 |
|----------|----------|----------|
| 通讯录同步 | 读取成员/部门 | 同步组织架构和考勤 |
| 考勤打卡 | 读取打卡数据 | 同步考勤记录 |
| 审批流 | 读取审批数据 | 同步审批记录 |
| 消息推送 | 发送应用消息 | 推送简报和预警 |
| 群发消息 | 发送消息到成员 | 推送工资条 |

#### 步骤2：配置可信域名
在"设置" → "域名可信"中添加：
```
可信域名: weda.cloud.tencent.com
```

## 三、微搭连接器配置

### 3.1 创建企业微信连接器

#### 步骤1：进入连接器配置
1. 登录微搭控制台
2. 进入"外部数据源" → "企业微信"
3. 点击"新建连接器"

#### 步骤2：配置连接信息

```yaml
连接器标识: wechat_work
描述: 企业微信消息推送和数据同步
```

#### 步骤3：配置认证信息
```yaml
认证方式: 企业微信应用
企业ID: wwxxxxxxxxxxxxxxx
AgentId: 1000001
Secret: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

#### 步骤4：测试连接
点击"测试连接"验证配置是否正确。

### 3.2 配置数据同步方法

#### 方法1：获取考勤数据
```yaml
方法名称: getAttendance
请求方式: POST
请求路径: /cgi-bin/report/getdaylist
请求参数:
  - 名称: starttime
    类型: 数字
    必填: 是
    描述: 开始时间戳
  - 名称: endtime
    类型: 数字
    必填: 是
    描述: 结束时间戳

响应结构:
  errcode: 数字
  errmsg: 字符串
  results: 数组
```

#### 方法2：获取审批数据
```yaml
方法名称: getApprovalData
请求方式: POST
请求路径: /cgi-bin/corp/getapprovaldata
请求参数:
  - 名称: starttime
    类型: 数字
    必填: 是
  - 名称: endtime
    类型: 数字
    必填: 是

响应结构:
  errcode: 数字
  errmsg: 字符串
  applys: 数组
```

#### 方法3：发送应用消息
```yaml
方法名称: sendMessage
请求方式: POST
请求路径: /cgi-bin/message/send
请求参数:
  - 名称: touser
    类型: 字符串
    必填: 是
    描述: 接收成员（@all为全部）
  - 名称: msgtype
    类型: 字符串
    必填: 是
    描述: 消息类型（text/markdown/news）
  - 名称: content
    类型: 字符串
    必填: 是
    描述: 消息内容

响应结构:
  errcode: 数字
  errmsg: 字符串
```

## 五、考勤数据同步

### 5.1 考勤数据同步工作流

```
触发时间: 每日 06:00

步骤1：获取考勤数据
- 调用getAttendance方法
- 获取昨日考勤记录
- 包含：打卡时间、位置、状态

步骤2：处理考勤数据
- 统计出勤天数
- 计算迟到/早退次数
- 统计请假/外出天数

步骤3：写入考勤表
- 创建或更新考勤记录
- 计算考勤扣减金额

步骤4：关联绩效
- 考勤扣减金额 → 工资表attendance_deduct
```

### 5.2 考勤数据字段

| 企业微信字段 | 微搭字段 | 说明 |
|--------------|----------|------|
| userid | person_id | 员工工号 |
| name | person_name | 员工姓名 |
| date | work_date | 上班日期 |
| checkin_time | check_in_time | 签到时间 |
| checkout_time | check_out_time | 签退时间 |
| work_sec | work_duration | 工作时长(秒) |
| location | check_location | 打卡位置 |
| is_late | is_late | 是否迟到 |
| is_early | is_early | 是否早退 |

### 5.3 考勤扣减计算规则

```javascript
// 考勤扣减计算
function calculateAttendanceDeduct(attendance) {
  let deduct = 0;
  
  // 迟到/早退扣款
  if (attendance.is_late) {
    deduct += 50; // 每次迟到扣50元
  }
  if (attendance.is_early) {
    deduct += 30; // 每次早退扣30元
  }
  
  // 请假扣款（按天）
  if (attendance.leave_days > 0) {
    deduct += attendance.leave_days * (8000 / 22); // 日薪
  }
  
  return deduct;
}
```

## 十、常见问题

### Q1: 消息发送失败，错误码40014
**原因**: access_token无效或过期
**解决**:
1. 检查企业微信应用Secret是否正确
2. 确保access_token已刷新（每2小时刷新一次）
3. 在微搭连接器中已自动处理

### Q2: 考勤数据为空
**原因**: 企业微信未开启考勤或无打卡记录
**解决**:
1. 确认企业是否已开通企业微信考勤
2. 检查是否有员工打卡
3. 确认API权限已申请

### Q3: 工作台应用无法打开
**原因**: H5地址配置错误或域名未可信
**解决**:
1. 检查H5地址是否正确
2. 确认域名已加入可信域名
3. 检查应用可见范围

### Q4: 消息接收不到
**原因**: 成员不在应用可见范围内
**解决**:
1. 检查成员是否在应用可见范围
2. 检查成员关注状态
3. 检查消息发送对象是否正确
